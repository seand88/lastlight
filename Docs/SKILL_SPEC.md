# SKILL SPECIFICATION

## 1. Overview
The Skill system provides permanent character progression in LastLight. While equipment defines the "verbs" (the active buttons a player can press), Skills act as the "grammar," providing modifiers, triggers, conversions, and scaling to those verbs. 

Skills are deeply integrated with the tag system; they look for specific tags on an ability (e.g., `Projectile`, `Fire`, `DoT`) to decide when to apply their mutations. This allows for a "Proc City" design where a simple generator ability can be evolved into a complex multi-projectile attack through skill synergy.

## 2. Design Goals & Functional Requirements
- permanent character progression purchased with XP.
- Permanent alterations; do not lose XP or skills on death.
- Lifetime cap of 45 Skill Points (allowing 5 skills to be fully maxed).
- Skills never add new buttons; they only mutate existing abilities via tags.
- Permanent alterations to your character. The currency is XP.

## 3. Data Specification
Skills are defined in `Skills.json`. (Note: Exact JSON schema is pending full implementation details).

### 3.1 Skill Object (Proposed)
| Property | Type | Description |
| :--- | :--- | :--- |
| `id` | string | Unique identifier (e.g., `skill_frost_magic`). |
| `name` | string | Display name. |
| `tags` | array | The core tags this skill mutates or adds to the player. |
| `tiers` | array | List of 5 Tier Objects defining progression. |

### 3.2 Tier Object (Proposed)
| Property | Type | Description |
| :--- | :--- | :--- |
| `tier_index` | int | 1 through 5. |
| `enablement` | string | (Tier 1) The baseline mechanical change. |
| `choices` | array | (Tiers 2 & 4) List of mutually exclusive perks. |
| `scaling` | object | (Tiers 3 & 5) Numeric stat increases or mechanical upgrades. |

## 4. Technical Implementation
- **Mutation Engine:** The `AbilityManager` and `EffectProcessor` must query the player's active skill list whenever an ability is triggered.
- **Stat Aggregator:** Skill bonuses (e.g., "+15% Poison Damage") must be summed before the `EffectProcessor` calculates final values.
- **Add-on Projectiles:** The server must handle skill-spawned projectiles (like the 5th-shot elemental rule) as separate `SpawnBullet` events with the `ManaGenerated` flag set to zero.
- **Synchronization:** A player's full skill tree is sent via `SelfStateUpdate` (Private Tier) to the local client only.
- **Visual Results:** Skill effects (like extra projectiles or changed colors) are broadcast to other players via standard combat packets (`SpawnBullet`, `EffectEvent`). 
- **Validation:** The server enforces skill point caps and tier requirements; the client merely displays what has been authorized.
